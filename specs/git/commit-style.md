# Git commit style

- Use the imperative mood in the subject line ("Add feature" not "Added feature")
- Subject line ≤ 72 characters
- Separate subject from body with a blank line
- Use the body to explain *why*, not *what* — the diff shows the what
- Reference issue numbers in the footer when applicable (`Fixes #123`)
- Prefer one logical change per commit; squash fixup commits before merging
- Never amend published commits
