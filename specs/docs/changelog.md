# CHANGELOG.md conventions

- `.spek/CHANGELOG.md` is a running log of completed work, appended at the end of each session
- Entries are grouped under a date heading (`## YYYY-MM-DD`), newest first
- Each entry describes what changed and why — written for a future reader, not as a session narration
- Record the *why* alongside the *what* — the diff shows the what
- One entry per session is typical; split into sub-entries only if the session covered genuinely unrelated areas
- Do not duplicate the CHANGELOG with a retro narration — it should read like a concise commit message, not a journal
